# Curriculum Learning Guide (強化学習 カリキュラム学習手順書)

このドキュメントは、実機展開（Sim-to-Real）を見据えた極めてシビアな物理環境下において、PPOモデルを段階的かつ効率的に学習させるための「カリキュラム学習（Curriculum Learning）」の手順をまとめたものです。

---

## 🎯 最終目標環境 (Target Specifications)
モデルが最終的に適応すべき、実機と同等の過酷な物理環境パラメータです。
* **最大モータトルク:** `0.03 Nm` (実機モータの限界値)
* **初期角度の変動幅:** `[-0.1, 0.1] rad` (限界復帰角度 0.114 rad に肉薄する過酷なスタート)
* **センサノイズ (標準偏差):** 角度 `0.005`, 角速度 `0.05`, モータ回転数 `0.5`

初期状態から上記の全制約を適用すると、モデルが有意義な方策（Policy）を獲得する前にエピソードが連続して終了してしまいます。そのため、以下の3つのフェーズに分けて段階的な学習とモデルの引き継ぎ（Resume）を行います。

---

## 🎯 フェーズ1：初期学習フェーズ（緩和条件下での基本方策の獲得）
モータの出力を意図的に高く設定し、ノイズのない理想環境で「倒立状態を維持する」ための基本的なアルゴリズムを獲得させます。

### 変更するコード (`envs/cube_env.py`)
1. **最大モータトルクを10倍（余裕を持たせる）:**
   ```python
   self.max_motor_torque = 0.3
   ```
2. **センサノイズの無効化:**
   ```python
   self.noise_std_angle = 0.0
   self.noise_std_gyro = 0.0
   self.noise_std_motor = 0.0
   ```
3. **初期角度の制限（直立付近からのスタート）:**
   ```python
   initial_angle = self.np_random.uniform(low=-0.01, high=0.01)
   ```

### 実行手順
- `0: [Initialize]` で学習を開始。
- **目安:** 10万〜20万ステップ。テストスクリプト上で安定して直立姿勢を維持できれば完了。

---

## 🎯 フェーズ2：ロバスト性の向上（モータトルク制約と初期角度の適応）
基本方策を獲得したモデルに対して、モータ出力を本来の実機スペックに制限し、限界角度からの復帰能力を学習させます。

### 変更するコード (`envs/cube_env.py`)
1. **最大モータトルクを実機スペックに戻す:**
   ```python
   self.max_motor_torque = 0.03
   ```
2. **初期角度を実機想定の限界値まで拡大:**
   ```python
   initial_angle = self.np_random.uniform(low=-0.1, high=0.1)
   ```
3. **センサノイズは無効（0.0）のまま維持。**

### 実行手順
- `1: [Resume]` でフェーズ1の重みを引き継いで学習を再開。
- **目安:** 10万〜30万ステップ。出力が制限された状態でも、限界角度から最大トルクを利用して復帰できるようになれば完了。

---

## 🎯 フェーズ3：Sim-to-Real 最終学習フェーズ（センサノイズの導入）
最後に、実機のIMUセンサやモータエンコーダのブレを模したドメインランダマイゼーション（ガウシアンノイズ）を導入し、現実のノイズに対する耐性を学習させます。

### 変更するコード (`envs/cube_env.py`)
1. **ノイズを実機想定の標準偏差に設定:**
   ```python
   self.noise_std_angle = 0.005      # 角度ノイズ
   self.noise_std_gyro = 0.05        # 角速度ノイズ
   self.noise_std_motor = 0.5        # モータ回転数ノイズ
   ```
2. **その他（トルク0.03、初期角度[-0.1, 0.1]）は維持。**

### 実行手順
- `1: [Resume]` でフェーズ2の重みを引き継いで学習を再開。
- **目安:** 20万〜50万ステップ。ノイズが印加された状態でも微小な振動を起こさず、安定した倒立を継続できれば最終学習完了。

---

## 💡 トラブルシューティングと履歴管理
* **フェーズ移行時の学習の不安定化:**
  環境難度が上がるため、移行直後は一時的にエピソードの早期終了が増加する場合があります。数万ステップ経過後に方策が適応し始めるため、十分なステップ数を確保してください。
* **方策崩壊（Policy Collapse）が発生した場合:**
  学習が後退してしまった場合は、本プロジェクトのアーカイブシステムを活用し、前フェーズの安定したチェックポイントから再度学習をやり直してください。
